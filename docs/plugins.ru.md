# Плагины

Плагины — основной способ расширения osysHome. Каждый плагин добавляет поддержку нового протокола, устройства или функциональности.

---

## Как работает система плагинов

При запуске osysHome:
1. Сканирует папку `plugins/` — каждая подпапка считается плагином
2. Проверяет в базе данных, активен ли плагин
3. Динамически импортирует модуль и создаёт экземпляр главного класса
4. Вызывает `initialization()` и запускает фоновые потоки (если нужны)

Плагины, отмеченные как **неактивные**, пропускаются полностью.

---

## Управление плагинами

### Просмотр всех плагинов

Перейдите в **Admin → Modules**. На странице отображается список всех плагинов с:

- Именем и описанием
- Категорией (Devices, App, System)
- Версией
- Статусом активности

### Включение / отключение

1. Найдите плагин в списке
2. Нажмите переключатель рядом с плагином
3. **Перезапустите приложение** — изменения вступят в силу только после перезапуска

### Настройка плагина

У каждого плагина есть собственная страница настроек по адресу `/admin/<PluginName>`.

## Обзор плагинов

### Системные плагины (System)

Устанавливаются по умолчанию и обеспечивают базовую функциональность.

| Плагин | Описание |
|--------|----------|
| **Scheduler** | Выполнение задач по расписанию (Cron) |
| **Objects** | Управление объектами, классами и свойствами |
| **Modules** | Отображение списка плагинов |
| **Users** | Управление пользователями и правами |
| **Backup** | Резервное копирование базы данных |
| **Cleaner** | Очистка устаревших данных и кешей |
| **wsServer** | WebSocket-сервер для обновлений в реальном времени |
| **ConsoleMonitor** | Просмотр системных логов в браузере |
| **Docs** | Встроенная документация (этот сайт) |
| **Permissions** | Расширенное управление правами доступа |

### Плагины устройств (Devices)

| Плагин | Протокол | Что поддерживает |
|--------|----------|-----------------|

| **Bluetooth** | BLE | Bluetooth Low Energy датчики и устройства |
| **Modbus** | Modbus RTU/TCP | Промышленные счётчики и контроллеры |
| **OpenHASP** | MQTT | Панели управления на базе OpenHASP |
| **XiaomiHome** | Mi Home | Устройства Xiaomi через облако Mi Home |
| **YandexDevices** | Yandex Smart Home | Устройства умного дома Яндекса |
| **Keenetic** | Keenetic API | Маршрутизаторы Keenetic (информация о сети) |

## Настройка популярных плагинов

### MQTT

MQTT — универсальный протокол для умного дома. Большинство устройств (датчики, лампы, выключатели) поддерживают MQTT.

**Настройка:**
1. Перейдите в **Admin → Mqtt**
2. Укажите адрес брокера (например, `192.168.1.10`)
3. Порт (по умолчанию `1883`)
4. Логин и пароль (если брокер требует)
5. Нажмите **«Сохранить»** и перезапустите

**Добавление устройства:**

- Плагин автоматически создаёт объекты при получении данных с новых топиков
- Или вручную привяжите топик к свойству объекта в настройках Mqtt

---

### Zigbee2MQTT (z2m)

Позволяет подключить сотни устройств Zigbee (датчики движения, температуры, лампы Philips Hue, IKEA и т.д.) через координатор Zigbee и брокер Z2M.

**Требования:**

- Запущенный [Zigbee2MQTT](https://www.zigbee2mqtt.io/) 
- MQTT-брокер (Mosquitto)

**Настройка:**
1. Настройте плагин **Mqtt** (адрес брокера)
2. Перейдите в **Admin → z2m**
3. Укажите топик Z2M (по умолчанию `zigbee2mqtt`)
4. Плагин автоматически создаст объекты для каждого устройства

---

### Tuya

Поддержка устройств Tuya/Smart Life — лампы, розетки, увлажнители, роботы-пылесосы.

**Настройка:**
1. Получите `Client ID` и `Client Secret` в [Tuya IoT Platform](https://iot.tuya.com/)
2. Перейдите в **Admin → Tuya**
3. Введите учётные данные
4. Нажмите **«Синхронизировать устройства»** — система загрузит список ваших устройств

---

### Telegram Bot

Отправка уведомлений и управление домом через Telegram.

**Настройка:**
1. Создайте бота у [@BotFather](https://t.me/BotFather) и получите токен
2. Перейдите в **Admin → TelegramBot**
3. Введите токен бота
4. Укажите список разрешённых пользователей (Telegram ID)
5. Нажмите **«Сохранить»**

**Использование в автоматизациях:**
```python
# Отправить сообщение через Telegram
setProperty("TelegramBot.message", "Датчик движения сработал в прихожей!")
```

---

### Google TTS / Yandex TTS

Синтез речи — озвучивание уведомлений через динамики.

**Google TTS:**
1. Установка не требует ключей API — используется бесплатный Google Translate TTS
2. Перейдите в **Admin → GoogleTTS**
3. Укажите устройство вывода (IP колонки или путь к аудио-выходу)
4. Выберите язык (по умолчанию `ru`)

**Использование:**
```python
say("Доброе утро!")
say("Температура в комнате " + str(getProperty("RoomSensor.temperature")) + " градусов")
```

## Зависимости плагинов

Некоторые плагины требуют дополнительных Python-библиотек. Они перечислены в файле `plugins/<PluginName>/requirements.txt`.

Установка зависимостей плагина:

```bash
pip install -r plugins/GoogleTTS/requirements.txt
pip install -r plugins/Mqtt/requirements.txt
```

Или установите все зависимости сразу (если поддерживается скриптом):

```bash
# Вручную для каждого нужного плагина
pip install -r plugins/Tuya/requirements.txt
pip install -r plugins/z2m/requirements.txt
```

## Установка нового плагина

Плагины распространяются как папки с кодом:

1. Поместите папку плагина в `plugins/`
2. Убедитесь, что в папке есть `__init__.py` с классом, имя которого совпадает с именем папки
3. Установите зависимости плагина (если есть `requirements.txt`)
4. Перезапустите osysHome
5. Плагин появится в **Admin → Modules** — включите его и настройте
