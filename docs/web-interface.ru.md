# Веб-интерфейс

osysHome управляется через браузер. Веб-интерфейс открывается по адресу `http://your-server:5000` (порт задаётся в `config.yaml`).

---

## Структура интерфейса

После входа в систему вы увидите навигационное меню с разделами:

| Раздел | URL | Описание |
|--------|-----|----------|
| **Dashboard** | `/` | Главная страница с виджетами устройств |
| **Objects** | `/admin/Objects` | Управление объектами и классами |
| **Modules** | `/admin/Modules` | Список всех плагинов |
| **Scheduler** | `/admin/Scheduler` | Расписание и задачи автоматизации |
| **Users** | `/admin/Users` | Управление пользователями |
| **Backup** | `/admin/Backup` | Резервное копирование |
| **Logs** | `/admin/Logs` | Просмотр журналов системы |
| **Docs** | `/docs` | Документация (этот сайт) |

Каждый плагин добавляет свой пункт меню в соответствующей категории.

---

## Dashboard

Dashboard — это главная страница с настраиваемыми виджетами.

### Виджеты

Виджеты добавляются плагинами, которые объявляют действие `widget`. Стандартные виджеты:

- **Текущее время и дата**
- **Присутствие** (кто дома)
- **Состояние устройств** (лампы, термостаты и т.д.)
- **График истории** значений свойств
- **Предстоящие задачи** из Scheduler

### Настройка Dashboard

Порядок и набор виджетов настраивается в разделе **Admin → Dashboard**. Виджеты можно перетаскивать и скрывать.

---

## Раздел Objects

Здесь вы управляете всей иерархией устройств.

### Просмотр объектов

- Список всех объектов с их классами и описаниями
- Кликните на объект, чтобы увидеть его свойства и текущие значения
- История изменений доступна для свойств с включённой записью истории

### Создание объекта вручную

1. Перейдите в **Admin → Objects**
2. Нажмите кнопку **«Добавить объект»**
3. Введите имя, выберите класс, добавьте описание
4. Нажмите **«Сохранить»**

После создания объекта можно добавить к нему свойства и методы.

### Редактирование свойств

На странице объекта можно:

- Добавить новое свойство (кнопка **«+ Свойство»**)
- Вручную изменить текущее значение свойства
- Включить запись истории
- Привязать метод к свойству (вызов при изменении)

### Редактирование методов

Методы — это Python-код, который выполняется внутри системы. Редактор кода доступен прямо на странице объекта.

Встроенные функции, доступные в методе:

```python
# Получить/установить свойство любого объекта
value = getProperty("SomeSensor.temperature")
setProperty("SomeLamp.state", True)

# Вызвать метод другого объекта
callMethod("SomeLamp.toggle")

# Произнести текст
say("Текст сообщения")

# Запустить звук
playSound("alert.mp3")
```

### Классы

Вкладка **«Классы»** показывает все шаблоны устройств. Здесь можно:

- Создать новый класс
- Добавить свойства и методы на уровне класса
- Задать шаблон (HTML/Jinja2) для отображения объектов этого класса

## Раздел Modules (Плагины)

Страница `/admin/Modules` показывает все зарегистрированные плагины.

Для каждого плагина отображается:

- **Имя и описание**
- **Категория** (Devices, App, System)
- **Версия**
- **Статус** (активен / неактивен)
- Кнопка для перехода в настройки плагина

### Включение/отключение плагина

1. Перейдите в **Admin → Modules**
2. Найдите нужный плагин
3. Нажмите переключатель активности
4. **Перезапустите систему** — изменение вступит в силу только после перезапуска

---

## Раздел Users

Управление пользователями системы.

### Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Admin** | Полный доступ ко всем настройкам |
| **User** | Просмотр Dashboard, ограниченное управление |

### Создание пользователя

1. Перейдите в **Admin → Users**
2. Нажмите **«Добавить пользователя»**
3. Введите логин, пароль, выберите роль
4. Нажмите **«Сохранить»**

---

## Раздел Logs

Централизованный просмотр журналов работы системы.

- Каждый плагин пишет в свой лог-файл (`logs/<PluginName>.log`)
- Общий лог системы: `logs/main.log`
- Лог ошибок: `logs/errors.log`
- Фильтрация по плагину, уровню (INFO, WARNING, ERROR) и времени

---

## История изменений

Исторические значения доступны для свойств, у которых `history > 0` (срок хранения в днях).
Визуализация и расширенный анализ выполняются через опциональные плагины, документация по ним находится в директориях самих плагинов.

## REST API

osysHome предоставляет REST API для интеграции с внешними системами.

### Базовый URL

```
http://your-server:5000/api/
```

### Основные эндпоинты

```http
GET  /api/objects              — список всех объектов
GET  /api/objects/{name}       — данные конкретного объекта
GET  /api/objects/{name}/{prop} — текущее значение свойства
POST /api/objects/{name}/{prop} — установить значение свойства

GET  /api/classes              — список классов
```

### Пример запроса (curl)

```bash
# Получить значение свойства
curl http://localhost:5000/api/objects/LivingRoomLamp/state

# Установить значение
curl -X POST http://localhost:5000/api/objects/LivingRoomLamp/state \
     -H "Content-Type: application/json" \
     -d '{"value": true}'
```

### Аутентификация

По умолчанию API доступен только для авторизованных пользователей. При работе из внешних систем используйте токен сессии или настройте отдельные права доступа.

## WebSocket

Dashboard получает обновления в реальном времени через WebSocket-соединение. Плагины могут отправлять данные в браузер вызовом:

```python
self.sendDataToWebsocket("update", {"object": "LivingRoomLamp", "property": "state", "value": True})
```

Это позволяет мгновенно отображать изменения состояния устройств без перезагрузки страницы.
