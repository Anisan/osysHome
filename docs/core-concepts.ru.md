# Основные концепции

В основе osysHome лежит объектно-ориентированная модель. Понимание пяти ключевых сущностей позволит вам эффективно работать с системой: создавать устройства, управлять ими и писать автоматизации.

---

## Обзор модели

```
Класс (Class)
  └── Объект (Object)         ← конкретное устройство
        ├── Свойство (Property)  ← параметр устройства
        │     └── Значение (Value)  ← текущее значение
        └── Метод (Method)       ← исполняемый Python-код
```

Вся эта иерархия хранится в базе данных и отражает реальный парк устройств вашего умного дома.

---

## Класс (Class)

**Класс** — это шаблон (тип устройства). Например: «Лампа», «Термометр», «Розетка».

- Классы определяют, какие свойства и методы есть у объектов этого типа.
- Один класс может быть **родительским** для другого (наследование).
- Плагины создают классы программно при инициализации.

**Пример классов:**

| Имя класса | Описание |
|------------|----------|
| `Lamp` | Умная лампочка |
| `Thermometer` | Датчик температуры |
| `Switch` | Выключатель |
| `Presence` | Датчик присутствия |

---

## Объект (Object)

**Объект** — это конкретный экземпляр класса, то есть реальное физическое (или виртуальное) устройство в вашем доме.

- Каждый объект принадлежит классу и наследует его свойства и методы.
- Объект можно создать через веб-интерфейс или программно из кода плагина.

**Пример:**

| Имя объекта | Класс | Описание |
|-------------|-------|----------|
| `LivingRoomLamp` | `Lamp` | Лампа в гостиной |
| `BedroomSensor` | `Thermometer` | Датчик температуры в спальне |
| `KitchenSwitch` | `Switch` | Выключатель на кухне |

---

## Свойство (Property)

**Свойство** — это именованный параметр объекта, который хранит значение.

Свойства могут быть:

- Определены на уровне **класса** — наследуются всеми объектами этого класса
- Определены на уровне **объекта** — уникальны для конкретного устройства

### Типы свойств

| Тип | Константа | Описание |
|-----|-----------|----------|
| Не задан | `PropertyType.Empty` | Универсальный тип |
| Строка | `PropertyType.String` | Текстовое значение |
| Число | `PropertyType.Integer` | Целое число |
| Дробное | `PropertyType.Float` | Число с плавающей точкой |
| Булево | `PropertyType.Boolean` | `True` / `False` |
| JSON | `PropertyType.Json` | Объект или массив |

### Параметры свойства

| Параметр | Описание |
|----------|----------|
| `name` | Уникальное имя свойства в рамках объекта/класса |
| `description` | Человекочитаемое описание |
| `history` | Сколько дней хранить историю изменений (0 = не хранить) |
| `method_name` | Имя метода, который вызывается при изменении значения |

---

## Значение (Value)

**Значение** — это текущее состояние свойства. Каждое изменение фиксируется с меткой времени.

- Хранится в таблице `Value` в базе данных
- Записывается через пакетный писатель (`BatchWriter`) для производительности
- История (таблица `History`) сохраняется за заданный период, если `history > 0`

---

## Метод (Method)

**Метод** — это блок Python-кода, привязанный к объекту или классу.

Методы запускаются:

- Автоматически при изменении значения свойства (если свойство настроено на вызов метода)
- Вручную через API: `callMethod("ObjectName.methodName")`
- Из кода другого метода или автоматизации

### Контекст выполнения

Внутри кода метода доступны специальные переменные:

| Переменная | Описание |
|------------|----------|
| `object` | Текущий объект (можно читать/менять его свойства) |
| `value` | Новое значение (при вызове из триггера свойства) |
| `old_value` | Предыдущее значение |
| `args` | Дополнительные аргументы |

**Пример кода метода:**

```python
# Метод "onTemperatureChange" у объекта BedroomSensor
if value > 25:
    say("В спальне жарко: " + str(value) + " градусов")
    setProperty("BedroomFan.state", True)
```

## Публичный API

Для работы с объектами из кода методов, плагинов и автоматизаций используется API из модуля `app.core.lib.object`:

### Чтение и запись свойств

```python
from app.core.lib.object import getProperty, setProperty

# Получить значение свойства
temp = getProperty("BedroomSensor.temperature")

# Установить значение свойства
setProperty("LivingRoomLamp.state", True)
setProperty("Thermostat.target_temperature", 22.5)
```

### Вызов методов

```python
from app.core.lib.object import callMethod

callMethod("LivingRoomLamp.toggle")
callMethod("Alarm.activate", {"reason": "motion detected"})
```

### Создание объектов программно

```python
from app.core.lib.object import addClass, addObject, addClassProperty
from app.core.lib.constants import PropertyType

# 1. Создать класс
addClass("MySensor", description="Мой датчик")

# 2. Добавить свойство к классу
addClassProperty(
    name="temperature",
    class_name="MySensor",
    description="Температура",
    type=PropertyType.Float,
    history=30  # хранить историю 30 дней
)

# 3. Создать объект
addObject("KitchenSensor", class_name="MySensor", description="Датчик на кухне")

# 4. Записать значение
setProperty("KitchenSensor.temperature", 21.3)
```

### Вспомогательные функции (`app.core.lib.common`)

```python
from app.core.lib.common import say, playSound, runCode

# Произнести текст (через TTS-плагин)
say("Добро пожаловать домой!")

# Воспроизвести звук
playSound("notification.mp3")

# Выполнить произвольный Python-код
runCode("setProperty('Light.state', True)")
```

## Пример: полный жизненный цикл устройства

```
1. Плагин z2m обнаруживает датчик Zigbee
        ↓
2. Создаёт класс "ZigbeeSensor" (если не существует)
        ↓
3. Создаёт объект "Hallway_Motion" с классом "ZigbeeSensor"
        ↓
4. Устанавливает свойство "occupancy" = True
        ↓
5. ObjectManager сохраняет значение в БД
        ↓
6. ObjectManager вызывает привязанный метод (если есть)
        ↓
7. Все плагины с action="proxy" получают уведомление
        ↓
8. Dashboard обновляет виджет в браузере через WebSocket
```

## Именование объектов и свойств

Соглашения по именованию:

- Имена объектов и классов: `CamelCase` или `snake_case` (без пробелов)
- Для доступа к свойству используется нотация с точкой: `ObjectName.propertyName`
- Имена чувствительны к регистру: `lamp.state` ≠ `lamp.State`

```python
# Правильно
setProperty("LivingRoomLamp.state", True)
getProperty("BedroomSensor.temperature")

# Неправильно — пробелы в имени объекта не поддерживаются
setProperty("Living Room Lamp.state", True)  # ошибка
```
